Coaching AI Tools for Coding, Data & Open Records

Charles Minshew · The Atlanta Journal-Constitution

NICAR 2026 · Thursday, March 5 · 11:30 a.m.

The ground rules

Do use AI tools for…

  • Writing & debugging code
  • Interpreting sample data
  • Drafting public records requests
  • Internal documentation
  • Understanding unfamiliar data formats
  • Taking better notes for yourself
  • As a super-powered search engine

Don't use AI tools for…

  • Final data analysis work
  • Handling sensitive personal data
  • Processing entire datasets
  • Writing stories or content for the public
  • Replacing the work of other team members
  • Making editorial decisions

Important: These guidelines are suggestions. Consult with your editors or newsroom attorney for policies specific to your organization.

After NICAR: What to try first

1. Draft a records request

Pick an agency you cover. Start with a specific address or entity. Ask AI to draft the request under your state's public records law. Then iterate: expand the scope, ask for structured data, and request specific field names.

2. Understand a new dataset

Upload a sample (not the full file) to ChatGPT or Claude. Ask for a data dictionary. Ask "What questions could a reporter ask with this data?" Then ask for code to explore it in R, Python, or Excel.

3. Debug or write code

Paste your error message along with your code. Describe what you expected vs. what happened. Ask it to explain the fix — not just give you the fix. You'll learn faster and catch AI mistakes more easily.

Prompt patterns that work

These are reusable prompt structures you can adapt for almost any task. The key insight: be specific, assign a role, and iterate.

Role + Task

You are a data reporter covering transportation in Indianapolis. Help me write an APRA request to IndyGo for bus inspection records system-wide.

Assigning a role helps the AI calibrate tone, detail level, and domain knowledge.

Show me, then explain

Write an R script that reads this CSV, calculates a rolling 7-day average of daily counts, and plots it. Then explain each step so I can modify it.

Gets you working code AND understanding. Essential for learning.

Iterate & expand

Good, now expand this request to cover all IndyGo locations, not just the transit center. Include Red Line and Purple Line BRT vehicles. Also ask for data in spreadsheet format.

Start narrow, then broaden. Each round builds on the last.

Ask for structured output

Give me this data as a CSV with columns for: date, location, inspection_type, result, inspector_name, deficiencies_noted.

Specifying the output format prevents vague, narrative-style answers.

Sanity check

What assumptions are you making in this analysis? What could be wrong with this approach? What would a skeptical editor ask about these numbers?

AI won't challenge itself unless you ask. Build this into your workflow.

Data dictionary

Here's a sample of 20 rows from this dataset. Write a data dictionary explaining each column: what it contains, the data type, and any patterns or issues you notice.

Upload a sample — never the whole dataset. AI is great at this.

The records request workflow

This was our main hands-on exercise. Here's the progression you can repeat for any state and any agency.

Step 1: Be specific

I want to write a records request for bus inspection
and safety reports at the Julia M. Carson Transit
Center, 201 E. Washington St., Indianapolis, IN.

The AI drafts a formal request citing Indiana's APRA (Ind. Code § 5-14-3-1 et seq.), specifies the address, and lists categories of records.

Step 2: Iterate and expand

You are a reporter looking at bus safety records for
IndyGo in Indianapolis. I want to request data
system-wide, not just one location. Include the
Red Line and Purple Line BRT vehicles.

Now the request covers the entire transit system — fleet-wide inspection data, near-miss analysis records, and system-wide safety assessments.

Step 3: Ask for structured data

Also add a paragraph requesting any available
database or spreadsheet with fields like: vehicle ID,
route assignment, date of inspection, type, result,
inspector name, and deficiencies noted.

This is the power move. Documents are useful, but structured data is what you need for analysis. Always ask for it explicitly.

Step 4: Verify everything

Before you send: verify the statute citation is correct for your state, confirm the agency name and address, check that the date range makes sense, and make sure your contact information is included. AI gets you 80% of the way there — the last 20% is your expertise.

Tools we discussed

We used ChatGPT for hands-on exercises, but these prompt patterns work across AI tools. Each has strengths worth knowing:

ChatGPT (OpenAI) — Widely available, good at code and conversation. Free tier is useful; paid tiers add file uploads and longer context. Claude (Anthropic) — Strong at long documents, careful reasoning, and nuanced writing. Gemini (Google) — Integrates well with Google Workspace. GitHub Copilot — Best for inline code suggestions if you're already in VS Code.

The prompting principles are the same across all of them. Pick the one your newsroom allows and get comfortable with it.
