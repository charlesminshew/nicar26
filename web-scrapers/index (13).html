Building Web Scrapers with ChatGPT and Claude

Charles Minshew · The Atlanta Journal-Constitution

NICAR 2026 · Friday, March 6 · 3:45 p.m.

Why scrape on the fly?

🚨 Breaking news

There's been an incident and you need to preserve public information from a website before it changes or disappears. School staff directories, government pages, court records.

📊 No download button

A reporter found data online but there's no export option. They need the data once for a story. You don't need a production scraper — you just need the data.

🔧 Starting from zero

You need to build a scraper you'll use again, but you don't know where to begin. AI can get you a working draft in minutes that you can refine over time.

The ground rules

✅ Good scraping practice

  • Scrape public data that's meant to be accessed
  • Be gentle — add delays between requests
  • Check for an API before scraping HTML
  • Save raw data before cleaning it
  • Document what you scraped, when, and from where
  • Review all AI-generated code before running it

🚫 Watch out for…

  • Scraping behind login walls or paywalls
  • Ignoring robots.txt without good reason
  • Hammering a server with rapid-fire requests
  • Running AI-generated code without reading it first
  • Assuming scraped data is clean or complete
  • Skipping the "is there a download button?" check

Important: These guidelines are just suggestions. Consult with your bosses or attorney for more specific recommendations.

Approach 1: HTML table scraping

This is the simplest case. The data is already in an HTML table on the page, and you can see it in the page source. Our example: passenger load data from the Columbia Regional Airport.

Step 1

Identify your data target

What data do you need? Where does it live? Is it on one page or spread across many? For COU, passenger data spans 2002–2026 across 25 separate pages.

Step 2

Inspect the page

Right-click → Inspect. Look at the HTML source. Is the data in a <table> tag? Is it loading dynamically via JavaScript, or is it all in the source? For COU, it's a static HTML table — the data is right there in the page source.

Step 3

Study the URL structure

Navigate between years and watch the URL change. COU uses ?ParamYear=2025 — that means you can loop through years by changing a number. This is your pagination strategy.

Step 4

Save the HTML source

View Source (Ctrl+U / Cmd+U) and save the page as a text file. This gives the AI the full HTML structure to work with — much better than describing the table yourself.

Step 5

Prompt your AI tool

Give it everything: the URL pattern, the year range, and the saved HTML. Let it write the scraper. See the prompt template below.

Example prompt for HTML scraping

HTML table scraper

I want to write a script that will collect data from this web page that has a table with passenger load data for the Columbia Regional Airport.

The page is an HTML table. The original URL of the table for 2002 is https://www.flycou.com/passenger-load-data/?ParamAirline&ParamYear=2002 and the data continues until 2025.

I'm copying and pasting the HTML of that first page for review.

[paste HTML source here]

Attach or paste the saved HTML file. The AI will identify the table class, column headers, and build the parser for you.

Approach 2: Finding hidden APIs

Sometimes the data isn't in the HTML at all — the page loads it from a backend API using JavaScript. The browser's Network tab reveals these hidden data sources. Our example: the Georgia Ethics Commission campaign finance search.

Step 1

Open DevTools before you interact

Right-click → Inspect → Network tab. Filter by Fetch/XHR to cut the noise. Now interact with the page — search, filter, paginate — and watch the requests appear.

Step 2

Find the data request

Look for the request that returns your data. On the GA Ethics site, it's called GetTransactionDetails. Click on it to see the details.

Step 3

Copy the payload

Click the Payload tab. This shows the JSON the page sends to the API. Every empty field is a filter you can fill in. pageNumber and pageSize control pagination.

Step 4

Check the response

Click the Response tab. This is the structured JSON data coming back. Look for a totalItems field — it tells you how much data exists. Copy one record as a sample.

Step 5

Copy the endpoint URL

Click the Headers tab. The Request URL is the API endpoint. Copy it. You now have everything you need to prompt the AI.

Example prompt for API scraping

API-based scraper

I found an API endpoint that returns Georgia campaign finance contribution data as JSON.

Endpoint: https://api-recordsearch.ethics.ga.gov/api/PublicTransactionDetails/GetTransactionDetails

Here's the payload it expects:
[paste payload JSON]

Here's a sample response record:
[paste one record from the response]

Write me a Python script that pulls all contributions between 2024-01-01 and 2024-12-31 and saves them to a CSV file. Handle pagination using the pageNumber field.

Give the AI all three pieces: the endpoint URL, the payload structure, and a sample response. It will figure out the rest.

Testing with curl

Before writing a full script, you can test the API directly from your terminal. This confirms the endpoint works and helps you understand the response structure.

curl -s -X POST \
  "https://api-recordsearch.ethics.ga.gov/api/\
PublicTransactionDetails/GetTransactionDetails" \
  -H "Content-Type: application/json" \
  -d '{
    "pageNumber": 1,
    "pageSize": 5,
    "sortBy": "Transaction Date",
    "sortType": "desc",
    "transactionTypeCode": "TCON",
    "filerName": "",
    "sourceName": "",
    "transactionAmountMax": null,
    "sourceTypeCode": "",
    "committeeType": "",
    "electionID": "",
    "reportName": "",
    "toDate": "2024-12-31",
    "fromDate": "2024-01-01",
    "byState": "",
    "electionType": "",
    "electionYear": "",
    "filerRegistrationGuid": null
  }' | python3 -m json.tool

Use | tail -10 instead of json.tool to jump straight to the totalItems count at the bottom of the response.

Two approaches compared

🌐 HTML Table Scraping

  • Data is visible in the page source
  • Pagination via URL parameters
  • Parse with BeautifulSoup or rvest
  • What you see is what you get
  • Best for: static pages, simple tables

🔌 API Calls

  • Data loads via JavaScript in the background
  • Pagination via payload fields
  • Response is structured JSON
  • Often includes fields not shown on the page
  • Best for: search interfaces, dynamic sites

Prompt patterns for scraping

Reusable prompt structures you can adapt for any scraping task.

Reconnaissance

Here's the HTML source of a page with data I need to scrape. Identify the table structure, column headers, and suggest the best approach to extract this data. [paste HTML]

Start here when you're not sure what you're looking at.

Pagination handler

This script works for one page. The URL pattern is [URL]?page=1 and there are 25 pages. Modify it to loop through all pages with a 2-second delay between requests and combine the results.

Always add a delay. Be kind to servers.

Error handling

Add error handling to this scraper: retry failed requests up to 3 times with exponential backoff, log any pages that fail, and save progress after each page so I don't lose data if it crashes.

Essential for large scraping jobs. Ask for this after the basic script works.

Debug help

This scraper returns empty results. Here's my code and the HTML of the page I'm trying to scrape. What's wrong? [paste code and HTML]

Give the AI both the code and the target HTML so it can compare selectors.

Format conversion

I have this working Python scraper but I need it in R using rvest. Convert it and explain the key differences.

Useful when your team works in a different language.

Data cleaning

I scraped this data but the numbers have commas and dollar signs, dates are inconsistent, and some cells have asterisks. Write a cleaning script that standardizes everything and outputs a clean CSV.

Scraping is only half the job. Always budget time for cleaning.

What to tell the AI (checklist)

The more context you give, the better the output. Before prompting, gather these:

📋 For HTML scraping

  • The URL (and how it changes between pages)
  • The saved HTML source of one page
  • What columns/fields you want to extract
  • The year range or page range to cover
  • Your preferred language (Python, R, etc.)
  • Desired output format (CSV, JSON, etc.)

📋 For API scraping

  • The endpoint URL (from the Headers tab)
  • The full request payload (from the Payload tab)
  • A sample response record (from the Response tab)
  • The totalItems count for your query
  • Any filters you want to apply
  • Desired output format (CSV, JSON, etc.)

Common gotchas

Gotcha

The AI scrapes the sample, not the site

If you paste HTML into the prompt, the AI might just extract data from that sample instead of writing a scraper. Be explicit: "Write a script that fetches and parses the page — don't just extract from the sample I gave you."

Gotcha

JavaScript-rendered content

If View Source shows empty tables but the page displays data, it's loading via JavaScript. Check the Network tab for API calls. If there's no API, you may need Selenium or Playwright — ask the AI to use those instead.

Gotcha

The scraper works once, then breaks

Websites change. Save your raw HTML/JSON responses alongside the parsed data. If the scraper breaks later, you'll still have the data you already collected.

Gotcha

Rate limiting and blocking

If you're getting 403 or 429 errors, you're probably hitting the server too fast. Add delays between requests. Start with 2 seconds and increase if needed. Some sites require headers like User-Agent — ask the AI to add appropriate headers.
